# Visualisation

For the educational and interactive requirement given by the client, a method of giving visual feedback, which represents the sound heard, should be given to the user. This visualization had to be both real-time and easy to read.

## Design Description

The PyQT library has been chosen to be used for the real-time plotting. This has been chosen due to its ability to show real-time plots and the ease of creating a GUI with it, which makes testing more efficient.

The signal processing group of the project handles the waveform generation, which is given in the form of chunks of 512 samples with a sample frequency of 44.1 kHz. To form a smooth wave representation, each chunk will be added into a larger chunk of 1024 samples, then shifted to the left so that another chunk can be added.

Three different ways of visualizing the sound are being used.

The first is the waveform, represented in a time-versus-amplitude graph. This is used to show the waveform, which can be compared to the sound heard by the students.

The second visualization is the Fourier transform of this wave. This Fourier transform is evaluated using the `fft()` function provided by the SciPy library. This function performs the discrete Fourier transform:

$$
y[k] = \sum_{n=0}^{N-1} e^{-2\pi j \frac{kn}{N}} x[n]
$$

The third visualization used is the waterfall spectrogram. This waterfall is created using the `scipy.signal` waterfall function. However, this function is only able to create a waterfall plot of a complete signal. Therefore, for this project, the waterfall is calculated only for a small chunk of the signal and then appended to the end of the waterfall display to create the illusion of a moving spectrogram.

Another visualization created is the note visualization. This visualization shows the notes that are being played by the theremin. For this, the frequency is transformed into MIDI notation via the following formula:

$$
n = 69 + 12 \log_2\!\left(\frac{f}{440}\right)
$$

The result of this formula is a number. To convert it to a note, we take the modulus of 12 of this number and use it as an index in an array of note labels. For the octave we divide the number by 12, subtract 1 and round it down to an integer:

```python
note_labels = ['C', 'C#', 'D', 'D#', 'E', 'F', 'F#', 'G', 'G#', 'A', 'A#', 'B']
note = note_labels[n % 12]
octave = n // 12 -1
```

To visualize these notes in a way to into a way that students without a musical background feel more familiar with has been done by representing the notes by viaulizing a piano and showing with key is played on it. To make this piano, simple white rectangles are drawn based on the minimal and the maximal midi value. The values chosen are:

```python
midi_min = 21 
midi_max = 108
```

These values are chosen according to the minimal and maximal midi values of a full piano.
