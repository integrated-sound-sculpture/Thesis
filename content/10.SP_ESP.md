# Microcontroller Subsystem

## introduction

The microcontroller will function as an input controller. The microcontroller will receive the following inputs:

1. Sensor oscillations
2. waveform selection switch
3. autotune switch
4. amplitude slider
5. calibration button

The microcontroller will process these inputs and will send the the amplitude, frequency and waveform-type (square, sine or triangle) to the microprocessor. The connection between the processor and the controller will be done using an UART protocol.

## sensor oscilations
The most important input of the microcontroller is the variable frequency block wave. The frequency of wave holds the information about the distance of the users hand. The capacitive sensor and the variable oscillator are designed by the other subgroup. The task of the ESP32 is to find the frequency of the blockwave and convert this frequency into a different frequency in the audible range. 

The input wave will have a frequency between 900 kHz and 950 kHz. This means that the pulse width will be around 1.1 ms. The ESP32 has a build-in pulse counter. The counter in combination with a timer can give the frequency of the blockwave. The ESP32 will read out the pulse register every 20 ms. After 20 ms the timer will be reset and the pulse register is cleared.



## pin layout
The pin layout for the ESP32 in our project can be seen in the following table.

| Subsystem | ESP32 Pin(s) | Purpose |
|:---------:|:------------:|:--------|
| Pulse Counter | GPIO 34 | Frequency generator input |
| Pulse Counter | GPIO 32 ↔ GPIO 35 | Internal pulse-count reference connection |
| Signal Generator | GPIO 25 | Triangle wave output |
| Signal Generator | GPIO 26 | Square wave or sine wave output |
| UART | GPIO 17 | TX (Transmit) |
| UART | GPIO 16 | RX (Receive) |
| Controls | GPIO 5 | Calibration button |
| Controls | GPIO 4 | Amplitude slider |
| Controls | GPIO 22, GPIO 23 | Rotary switch waveform selection |
| Controls | GPIO 12 | Autotune button |
