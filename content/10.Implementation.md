# Implementation and Verification of the Full System
## Introduction
All of the different submodules were connected to each other and tested as an entire system. When the antenna is not connected, the oscillator can generate a detectable wave around 900 kHz. The ESP32 will read this data and convert it to an output frequency. A control panel is used to change the amplitude and waveform of the output signal. This is all sent to the Raspberry Pi using a UART connection. The Raspberry Pi will build the audio wave, additionally it will drive a led array which functions as a playing guide. The waves and their frequencies are also displayed on an external screen.


{numref}`theremin1` shows the inside of the sound sculpture. The oscillator, ESP32 and Raspberry Pi can be seen. {numref}`theremin2` shows the outside of the sound sculpture. The user input control, the playing guide and the speaker can be seen.

## Results & Discussion
(subsec-frequency-measurements=)
### Frequency
Without the antenna the system works as expected. The data shown in the table {numref}`tab-impl_results` was collected using a function generator to mimic the oscillator. It was all recorded using an input frequency range from 520 to 610 kHz. The auto-tune button was switched on and the waveform that was selected was a sine wave.

```{table} Measured output frequency when connecting the sound sculpture to a function generator which produces waves between 520 and 610 kHz. Using a sine wave with auto-tune.
:name: tab-impl_results
:width: 40%

| input frequency (function generator) [Hz] | output frequency (audio) [Hz] |
|:-------------------:|:----------------------:|
| 520 000| 131.0 |
| 530 000| 139.1 | 
| 540 000| 155.3 |
| 550 000| 174.4 |
| 560 000| 195.9 |
| 570 000| 233.0 |
| 580 000| 294.4 |
| 590 000| 440.1 |
| 600 000| 740.0 |
| 610 000| 2100 |

```
The output frequency maps to frequencies between 131 Hz and 2100 Hz. This is a 4 octave frequency range. It can be seen in the table that the output frequency does not depend linearly on the input frequency. This is due to the logarithmic behavior of human hearing and due to the non-linear behavior of both the capacitance and of the oscillator. The goal of the mapping is to hear a linear change in output notes when the hand position is changed.

### Amplitude
The amplitude is measured using a potentiometer, this sensor gives a linear range from 0 to 4096 as an output. The amplitude of the wave can therefore be configured with high precision. The volume of the speaker that is used depends a lot on the frequency. The amplitude of the audio output will therefore change when the frequency is changed. 

### Delay
The delays of the microcontroller and the microprocessor are shown in the table {numref}`tab-impl_delay`. The speaker and the oscillator circuit add a negligible amount of delay so they were omitted in the calculations of the total delay. 

```{table} The maximum delay of each subsystems
:name: tab-impl_delay
:width: 40%

| subsystem | delay [ms] |
|:-------------------:|:----------------------:|
| Microcontroller    | 21 |
| Microprocessor without visualisation   | 40 | 
| Visualisation   | 68 | 
```


The audio of the sound sculpture reacts within 61 ms to a change in hand position and changes the shown waveform and frequency domain of the signal within 129 ms.

### Implementation with antenna
When the antenna (instead of a function generator) is connected to the system the capacitance between the hand and the antenna changes a lot. Even when the hand is held at the same place, the oscillations of the oscillator circuit varies significantly. The capacitance between the hand and the antenna is very small. Therefore the environment has a relative large impact on the behavior of the oscillator circuit. At these capacitance levels, the system also becomes susceptible to large amounts of noise.

The large errors make it very difficult to calibrate the system correctly. A wrongly calibrated system only has a small playable output range of only a few Hertz. Hence it became basically impossible to correctly calibrate and play the sound sculpture with a connected antenna instead of a function generator.

## Conclusion
The full sound sculpture system was successfully implemented and verified using a function generator as the oscillator input. The ESP32, Raspberry Pi, audio synthesis, visualisation, and user controls all operated as intended. The system  mapped input frequencies between 520 kHz and 610 kHz to a four-octave audio range from 131 Hz to 2100 Hz, while providing configurable amplitude and waveform selection. Measured delays showed that the audio output responded within 61 ms and the visual feedback responded within 129 ms to a user input. Both the audio and the screen have noticeable delays.

When the antenna was connected, the oscillator became highly sensitive to environmental influences and electrical noise. The resulting frequency instability made reliable calibration difficult and significantly reduced the playable range of the instrument.
