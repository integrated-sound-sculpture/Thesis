# Implementation and Verification of the Full System
## Introduction
All of the different submodules were connected to each other and tested as an entire system. When the antenna isn't connected, the oscillator can generate a detectable wave around 900 kHz. The ESP32 will read this data and convert it to an output frequency. A control panel is used to change the amplitude and waveform of the output signal. This is all send to the Raspberry Pi using a UART connection. The Raspberry Pi will build the audio wave additionally it will drive an led array which functions as a playing guide. The waves and their frequencies are also displayed on an external screen.

## Results & Discussion
### Frequency
Without the antenna the system works as expected. The following data was collected using a function generator to mimic the oscillator. It was all recorded using a input frequency range from 520 to 610 kHz. The auto-tune button was switched on and the waveform that was selected was a sine wave.

```{table} measured output frequency when connecting the theremin to an function generator which produces waves between 520 and 610 kHz. Using a sinewave with auto-tune.
|:-------------------:|:----------------------:|
| 520 000| 131.0 |
| 530 000| 139.1 | 
| 540 000| 155.3 |
| 550 000| 174.4 |
| 560 000| 195.9 |

```
As you can see the output frequency perfectly maps to frequencies between 131 Hz and 2100 Hz. This is a 4 octave frequency range. It can be seen in the table that the output frequency does not depend linearly on the input frequency. This is firstly explained by the logarithmic behavior of our hearing. Secondly it can be explained due to the non-linear behavior of both the capacitance and of the oscillator. The goal of the mapping is to hear a linear change in output notes when the hand position is changed.

### Amplitude
The amplitude is measured using a potentiometer, this sensor gives a linear range from 0 to 4096 as an output. The amplitude of the wave can therefore be configured with high precession. The volume of the speaker that is used depends a lot on the frequency. The amplitude of the audio output will therefore change when the frequency is changed. 


### Delay
The theremin reacts within 50 ms to a change from the oscillator and changes the shown waveform and frequency domain of the signal within 100 ms. This was measured by calculating the runtime of individual functions and adding them together. Although the reaction time does not feel instant, the delay is minimal and hardly noticeable.

### Measuring with antenna
When the antenna (instead of a function generator) is connected to the system the capacitance between the hand and the antenna changes a lot, even when the hand is held at the same place, the oscillations of the oscillator circuit will vary a lot. The capacitance between the hand and the antenna is very small. Therefore the environment has a relative large impact on the behavior of the oscillator circuit. At these capacitance levels, the system also becomes susceptible for large amounts of noise.

The larges errors makes it very difficult to calibrate the system correctly. A wrongly calibrated system only has a small playable output range of a couple of Hertz. Hence it became basically impossible to correctly calibrate and play the theremin with a connected antenna instead of a function generator.
