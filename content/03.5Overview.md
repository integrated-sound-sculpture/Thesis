# Full system overview

From the division of the functions the full system, the time constraints and the set up requirements for this system, a devision of subsystems can be made. The decision is made to implement a simple capacitive sensor circuit for the pitch control and read and process the frequency digitally for clean and highly adaptable sound output. The rest of the inputs are then either sliders, knobs, or buttons, which are read out using a simple ADC or direct digital inputs. The audio output is done using a DAC/audio card and then amplified before the speaker. The visual output is done over HDMI for adaptability and due to time constraints. The playing guide is an array of LEDs, digitally controlled using digital output pins. The full system is illustrated in [](#fig-Design). The Hardware & Sensing subsystem consists of providing the sensing/user input elements, power to the whole system, amplifying the output signal to a speaker and the visualisation. The signal processing subsystem consists of the digital processing done in the micro controller and microprocessor, except for the visualisation. The division is also illustrated in [](#fig-Design).

The devision of the subsystems also leads to interaction between each subsystem, [](#tab-interface) shows the interfaces between the subsystems with a short description of its purpose.

```{table}
:label: tab-interface
:caption: Interfaces between subsystems
| Description | From | To | Communication format |
| :--- | :--- | :--- | :--- |
| Pitch control using a sensor | Frequency generation | Microcontroller | Block wave with amplitude of 0-3.3V and a frequency between 400kHz and 800kHz. |
| Amplitude control using a slider | Volume control slider | Microcontroller | Variable voltage of 0-3.3V |
| Autotune setting and calibration input control | Autotune switch & calibration button | Microcontroller | Voltage of either 0 or 3.3V |
| Options for the sound and visualisation control | Waveform select & visualisation select | Microcontroller | Two pins with both a voltage of either 0 or 3.3V |
| Providing power for the device | Power Unit | Microcontroller, microprocessor, frequency generation, playing guide & audio amplifier | A voltage of 5V |
| Settings communication | Microcontroller | Microprocessor | UART serial commands: `"<frequency>, <amplitude>, <waveform>, <visualisation setting>"` and `"calibration"` [](#subsec-uart) |
| Audio output | Microprocessor | Audio amplifier | Analog audio signal |
| Visualisation samples | Microprocessor | Visualisation | Sample buffer array |
```
